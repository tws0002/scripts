2. ##############################################################################
Access by Index

你可以藉由 index 存取 list 中的個別物件 . Index 就像是 list 中物件的地址. Index 應該放在 list 名稱之後, 用 [ ] 符號框住, 像下面這樣: list_name[index].

List 的 index 從 0 開始, 而不是 1! 要存取第一個物件應該要像這樣輸入: list_name[0]. 以此類推, 第二個物件的 index 是 1: list_name[1]. 電腦科學家喜歡從 0 開始計算.

Slicing Lists and Strings

my_list[3:]

如果你想要包含 list 裡面的 第一個或是最後一個物件, 那冒號的前面或後面不要填入任何數字就可以了.

For One and All

for variable in list_name:
    # Do stuff!

for 後面的 variable 名稱; 指的是在 list 裡每一個物件, 每一次執行的 variable name.

然後 in list_name 指的是 list_name 會是在這個迴圈中使用的 list . 這一行最後以一個冒號 (:) 做結尾, 而下面縮排的程式碼內容則是 list 中每一個物件都會執行一次.

10. ##############################################################################
This Next Part is Key

Dictionary 和 list 很相似, 但是你是透過 key 來找它對應的 value, 而不是像 list 一樣用順序性的 index. key 可以是 string 或是 數字. Dictionary 是用 curly braces (大括號), 像下面這樣:

d = {'key1' : 1, 'key2' : 2, 'key3' : 3}

這是一個叫做 d 的 dictionary, 裡面有三組 key 和對應的 value. Key 'key1' 對應 value 1, 'key2' 對應 2, 以此類推.

Dictionaries 就像電話簿一樣好用 (名字對應電話號碼), 或是像網路上的登入頁面, (e-mail 對應使用者名稱)!

11. ##############################################################################
New Entries

就像 list 一樣, Dictionaries 是 mutable (可變的). 也就是說他們在被創造出來之後可以再修改. 這樣的好處是你創造一個 dictionary 之後可以在加入新的 key/value, 像下面這樣:

dict_name[new_key] = new_value

一對空的 curly braces {} 是一個空的 dictionary, 就像一對空的 [] 是一個空的 list 一樣.

len() 作用再 dictionary 上會找出它總共有多少組 key-value. 一組 key-value 算是一個, 即使 value 是一個 list 也還是和它對應的 key 算做'一'組. (Dictionary 的 value 可以是 list!)
